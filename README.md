# 週の予定表作成Webアプリ

小学校の週の予定表（時間割・連絡事項）をブラウザ上で編集し、Excel（.xlsx）として
ダウンロードできる、GitHub Pages配信向けの静的Webアプリです。
データはすべてブラウザ内（LocalStorage）で完結し、外部サーバーには送信されません。

## 使い方（ローカル確認）

`index.html` をブラウザで直接開くだけで動作します（ビルド不要）。

## ディレクトリ構成

```
index.html          画面のHTML本体
css/style.css        見た目のスタイル
js/
  config.js          教科マスタ（教科名⇔画像の対応）、表示日数などの設定値
  dateUtils.js        日付計算の小さなユーティリティ
  storage.js          LocalStorageへの保存・読み込みをまとめた層
  dataStore.js        予定データ・レイアウト本体の管理（マスタデータの保持・期間抽出）
  render.js           画面描画（表の組み立て・編集イベント・画像ドラッグ&ドロップ）
  subjectManager.js   「教科・イラストの管理」画面（名前変更・画像差し替え・追加削除）
  layoutManager.js    「予定表のレイアウト編集」画面（行の追加・削除・並べ替え）
  galleryUI.js        イラストギャラリーUIの共通部品（管理画面・イラストパレット共用）
  imageLibrary.js     イラスト素材の全件マニフェスト（自動生成・手動編集しない）
  excelExport.js       ExcelJS(CDN)を使ったExcel出力
  main.js             起動処理・ボタンのイベント配線
images/
  subjects/            道徳・生活・総合・学活用の仮アイコン（SVG。まだ対応イラストが無い教科）
  library/             イラスト素材集「ドロップス」から取り込んだ全1406点
                        （大分類/中分類のフォルダに仕分け。ファイル名は配布時の番号付きのまま）
templates/
  schedule_template.xlsx  参考Excelから作った雛形（表示日数がちょうど6日のときのExcel出力に使う。
                          実際にアプリが読み込むファイルはこれだけ）
reference/
  reference.xlsx        学校で実際に使っている参考Excel（デザインの元ネタ。アプリ動作には未使用）
review/                 出力したExcelで見つかった不具合を報告してもらうための一時置き場
                        （アプリ動作には未使用。調査用の参考データであり、他ディレクトリへは反映しない）
data/                  将来のインポート機能で使うサンプルデータ置き場（現状は空）
```

### イラスト素材について

`images/library/` には、外部で配布されているイラスト素材集「ドロップス」の全イラストを
そのまま取り込んであります（ファイル名も変更していません）。画面の「教科・イラストの管理」
から、大分類→中分類→検索でイラストを探し、好きな教科に割り当てられます。
教科名とイラストの対応づけは `js/config.js`（初期値）または管理画面での設定
（LocalStorageに保存）だけで行い、素材ファイル自体はリネームしません。

## データ構造の考え方

予定表の「行構成（レイアウト）」と「各日の中身」は分けて持っている。

**レイアウト**（`weeklySchedule_layout`）: どんな行がどの順番で並ぶかという設定。
「予定表のレイアウト編集」画面で先生が自由に追加・削除・並べ替えできる。

```js
[
  { id: "section_notice", label: "お知らせ", type: "text" },     // 自由記述（複数行）
  { id: "section_period1", label: "1", type: "subject" },        // 教科プルダウン+イラスト
  { id: "section_lunch", label: "給食", type: "check" },          // ○×
  // ...
]
```

**予定データ**（`weeklySchedule_masterData`）は「1日分のオブジェクト」の配列としてLocalStorageに
保存されている（曜日ごとに固定した形ではなく、日付をキーにした可変長の配列）。各日の中身は、
レイアウトの行idをキーにした`cells`に持つ。

```js
{
  date: "2026-07-20",      // YYYY-MM-DD
  dayOfWeek: "月",          // 日付から自動計算
  cells: {
    "section_notice": { type: "text", value: "遠足のお知らせ" },
    "section_period1": { type: "subject", subject: "kokugo", note: "", customImage: "" },
    "section_lunch": { type: "check", value: "○" },
  },
}
```

`subject`セルの`customImage`は、イラストパレットからドラッグ＆ドロップで割り当てた画像を
上書き表示するための項目（空なら教科の既定イラストを使う）。教科プルダウンで選び直すと
自動でクリアされる。

「担任より」のように、特定の日付にひもづかない週単位のメモは`cells`とは別に
`weeklySchedule_weekNotes`（表示中の週の先頭日付をキーにしたオブジェクト）に保存する
（`dataStore.getWeekNote(weekStartDate)` / `setWeekNote(...)`）。週が変われば別のメモになる。

「どんな行があるか（レイアウト）」と「各日の中身（cells）」を分けているので、レイアウトを
変えても既存データが壊れず、画面（render.js）とExcel出力（excelExport.js）は
どちらも同じレイアウトをループしてセルを出し分けるだけで済む。

画面には、予定データの配列の中から「今表示したい日付（5〜9日分）」だけを抽出して表示します
（`dataStore.getEntriesForDates`）。表示・Excel出力のロジックはこの抽出関数を通して
データを受け取るだけなので、将来Excel/PDF/Googleスプレッドシートから週・月・年単位の
データを一括インポートする機能を追加しても、既存の画面側コードを大きく変えずに済む設計です。

表示する日付は、列ごとに個別編集するのではなく「週を選択」（`js/render.js`の`selectWeek`）で
決める。選んだ日付が属する週の月曜日を自動で求め、月〜金＋翌週月曜の6日間を組み立て直す
（`前の週`/`次の週`ボタンも同様に週単位で移動する）。それ以外に見せたい日（土曜授業など）は、
右端の「＋ 追加」で日付を指定して個別に追加でき、5〜9日の範囲で×ボタンにより削除もできる。

## クラス名・印刷（A4）対応

画面上部の入力欄に入れたクラス名（`weeklySchedule_className`に保存）は、表示中の日付範囲と
組み合わせてタイトルとして表示され、Excel出力の1行目にも同じ内容が入る。

- 画面をそのまま印刷（Ctrl+P）すると、ボタン類やイラストパレットは印刷されず、
  A4横向きに収まるよう`css/style.css`の`@media print`で調整している。
- Excel出力（`js/excelExport.js`）は表示日数によって方式を切り替える。
  - **表示日数がちょうど6日**: `templates/schedule_template.xlsx`（参考Excelの構成をそのまま
    保った雛形。1日＝1列、日ごとの中身だけを空にしてある）を読み込み、`config.js`の既定
    レイアウトの行idと雛形の行番号を対応づけて値を書き込む。日付は実際のExcel日付値として
    書き込む（タイムゾーンでずれないようUTC基準で変換）。先生がレイアウトに独自の行を
    追加した場合は雛形の下に追記される。「担任より」（日付にひもづかない週メモ）は
    日の列をまたいで1セルに結合して書き込む。
  - **6日以外（5日・7〜9日）**: 雛形は6日固定の列構成のためそのまま使えない。かわりに雛形から
    フォント・罫線・列幅などの「見た目」だけを読み取り（`extractTemplateLook`）、日数に合わせて
    コードで組み立てる（1日＝1列）。
  - どちらの場合も`worksheet.pageSetup`でA4横・1ページ幅に収まる設定にしている。
  - `templates/schedule_template.xlsx`はExcelで直接編集してよい。行・列を大きく変更した場合は
    `js/excelExport.js`内の`TEMPLATE_ROW_MAP`/`TEMPLATE_DAY_COLS`/`TEMPLATE_WEEK_NOTE_ROW`の
    対応表を合わせて更新する必要がある。
